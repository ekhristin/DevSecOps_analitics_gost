## Рекомендации (минимальный обязательно)
1. Оформить регламент безопасной разработки ПО (SDLC).  
2. Вписать программные требования (C#, JS, SQL) + правила secure coding.  
3. Уточнить процесс трекинга уязвимостей / техдолга с SLA по критичности.  
4. Доделать threat modeling/attack surface для ключевых проектов.  
5. Добавить DAST/IAST/неконтрактный динамический анализ.  
6. Внедрить SCA/анализ IaC/контроль зависимостей (SBOM).  
7. Описать процесс работы с секретами и управление доступом в GitLab/Jenkins.  
8. Настроить процесс безопасной сборки и промежуточного хранения артефактов (CI/CD).  
9. Развернуть регламент release - приемки/поставки/поддержки.
